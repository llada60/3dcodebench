**###**

**SPLIT phase**

       { "name": "<GroupName>", "prefix": "<KeyPrefix_>",

         "symmetry": "mirror_x" | "none",

     ]
   }
2. Spawn ONE subagent per group via the Task tool, in parallel. Each subagent
   prompt must contain: (a) the absolute image paths listed above, with an
   instruction to Read all four images itself before modeling, (b) the full
   `conventions` and `global_bbox` from plan.json (copied verbatim, never
   re-derived), (c) only its own group spec, (d) the PRIMITIVE SCHEMA section
   below, copied verbatim.

   PRIMITIVE SCHEMA (open-ended)
   - `type` may be ANY parametric primitive you judge best for the local
     geometry: rounded_box, uv_sphere, cylinder, cone, torus, capsule, plane,
     ico_sphere, tube, wedge, extruded polygon, lathe/revolve profile, ... —

       `type`, `location` (global coords), `rotation` (XYZ Euler, radians),

**###**

**MERGE phase**

1. Read every `parts/*.json`. Fail loudly (re-dispatch that group once) if a
   file is missing, malformed, or violates the schema.
2. Resolve conflicts deterministically (type-agnostic):
   - Duplicate keys → keep first, rename later ones `<key>_2`, `<key>_3`...
   - `_L`/`_R` pairs must be exact mirrors across X; if they diverge, mirror
     L onto R (works for any type: negate x-location, mirror rotation, keep
     parameters identical).
   - World AABB of each part = rotate the local `bbox_size` box by
     `rotation`, translate by `location`. ALL spatial checks (ground contact,
     global_bbox fit, interface snapping, overlap detection) use this AABB
     only — never type-specific math.
   - Nothing below ground: world AABB zmin >= -0.005; clamp via Z translate.
   - Sanity: all size/radius/depth params > 0; bevel (if present)
     <= 0.5 * min(bbox_size); rotation values finite.
3. Snap interfaces declared in plan.json: e.g. `sits_on` means the lower
   part's AABB top and the upper part's AABB bottom differ by <= 0.01 —
   translate the upper part in Z to close the gap.
4. Validate the merged dict with a small Python script (`validate.py`):
   - required fields present on every part;
   - for types with known closed-form bounds, recompute the bbox and warn if
     the declared `bbox_size` deviates > 5% (unknown types accepted on
     trust);
   - merged AABB fits `global_bbox` (±10%); symmetry check; ground check.
   Run it with Bash and fix any reported issue, re-running until clean
   (max 3 iterations).
5. Write the final result as `parts_dict.py` containing a single
   `parts_dict = {...}` literal, formatted like the example. This file is the
   only deliverable; print a one-line summary (group count, part count,
   distinct types used).

Example final syntax (field values are illustrative only):

parts_dict = {
    "Body_main": {
        "type": "rounded_box",
        "location": (0.0, 0.0, 0.5),
        "rotation": (0.0, 0.0, 0.0),
        "bbox_size": (1.0, 0.6, 1.0),
        "bevel": 0.05,
    },
}

**###**

**Rules**

- Subagents never modify conventions, global_bbox, or other groups' files.
- A subagent may introduce any primitive type, but the downstream builder
  must be able to construct it from parameters alone. If a shape cannot be
  expressed parametrically, decompose it into multiple primitives instead of
  inventing a pseudo-type like "mesh" or "custom".
- The orchestrator never hand-edits geometry before validation passes on the
  raw merge; all fixes go through MERGE steps 2–4 so they are reproducible.
- If two groups overlap badly (>30% AABB volume intersection between parts of
  different groups not declared as an interface), shrink or move the smaller
  part, preferring translation along the interface normal.

Do not create files other than `plan.json`, `parts/*.json`, `validate.py`, and
`parts_dict.py`. Do not access or modify anything outside the current
workspace. The final `parts_dict.py` must contain exactly one assignment and no
imports, executable calls, helper variables, Markdown, or prose.
